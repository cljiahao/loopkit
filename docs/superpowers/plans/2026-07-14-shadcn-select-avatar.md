# shadcn Select + Avatar Conversion Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace two hand-rolled UI pieces (`ProgramSwitcher`'s native `<select>`, `DashboardNav`'s hand-rolled avatar `<span>`) with shadcn's `Select` and `Avatar` components, per the user's standing "always use shadcn if possible" rule.

**Architecture:** Install both shadcn components via the CLI in one shared setup step, then two independent migrations: `ProgramSwitcher`'s internals swap to `Select` while its public API (`programs`/`currentId`/`basePath`) stays byte-identical, so none of its three call-site pages need touching. `DashboardNav`'s avatar markup swaps to `Avatar`/`AvatarImage`/`AvatarFallback` inside the existing, untouched `DropdownMenuTrigger` button.

**Tech Stack:** Next.js 16 App Router, `radix-ui` (unified package, already a dependency — this repo's existing `dropdown-menu.tsx` imports from it as `import { DropdownMenu as DropdownMenuPrimitive } from "radix-ui"`), TypeScript strict, Tailwind v4, Vitest + Testing Library (jsdom), pnpm.

## Global Constraints

- Keep the codebase clean: no leftover hand-rolled `<select>` or avatar `<span>` markup coexisting with the new shadcn components; remove the `next/image` `Image` import from `dashboard-nav.tsx` (confirmed unused anywhere else in that file once the avatar swap lands).
- Every task's commit must leave `pnpm check` (prettier + eslint + tsc) clean.
- `ProgramSwitcher`'s public API (`programs`, `currentId`, `basePath` props) must not change. Zero edits to `src/app/dashboard/stats/page.tsx`, `src/app/dashboard/activity/page.tsx`, or `src/app/dashboard/customers/page.tsx`.
- The internal `"all"` sentinel must never leak into the URL: merged view still produces no `?p=` param, filtered view still produces `?p=<real-program-id>` — identical URL contract to today.
- `DashboardNav`'s `DropdownMenuTrigger` button wrapper and `src/components/ui/dropdown-menu.tsx` itself are not touched — only the avatar markup inside the trigger changes.
- This repo's shadcn config (`components.json`): style `new-york`, RSC on, Tailwind CSS at `src/app/globals.css`, base color `neutral`, CSS variables on, aliases `@/components`, `@/components/ui`, `@/lib/utils`. No `ui:add`/`shadcn` script in `package.json` — the bare CLI invocation `pnpm dlx shadcn@latest add <component>` is correct for this repo.

Replace the full contents of `src/app/dashboard/program-switcher.tsx`:

```tsx
"use client";

import { useRouter, useSearchParams } from "next/navigation";
import {
  Select,
  SelectContent,
  SelectItem,
  SelectTrigger,
  SelectValue,
} from "@/components/ui/select";

const ALL_PROGRAMS = "all";

// Same-page program switcher for every Stats/Activity/Customers view (merged
// and filtered alike), mirroring qkit's StatsControls: one instant picker,
// no submit button. Copies the current URL's other params (e.g. Customers'
// `q` search term) forward so switching programs never drops them. Uses
// shadcn's Select — Radix disallows an empty-string item value, so the
// "All programs" state (currentId === "") maps to an internal "all"
// sentinel that never leaks into the URL.
export function ProgramSwitcher({
  programs,
  currentId,
  basePath,
}: {
  programs: { id: string; name: string }[];
  currentId: string;
  basePath: string;
}) {
  const router = useRouter();
  const searchParams = useSearchParams();

  if (programs.length <= 1) return null;

  function handleChange(value: string) {
    const params = new URLSearchParams(searchParams.toString());
    if (value && value !== ALL_PROGRAMS) {
      params.set("p", value);
    } else {
      params.delete("p");
    }
    const query = params.toString();
    router.push(query ? `${basePath}?${query}` : basePath);
  }

  return (
    <Select value={currentId || ALL_PROGRAMS} onValueChange={handleChange}>
      <SelectTrigger
        aria-label="Switch program"
        className="mb-4 h-9 w-auto min-w-[10rem] rounded-lg border bg-card px-3 text-sm"
      >
        <SelectValue />
      </SelectTrigger>
      <SelectContent>
        <SelectItem value={ALL_PROGRAMS}>All programs</SelectItem>
        {programs.map((option) => (
          <SelectItem key={option.id} value={option.id}>
            {option.name}
          </SelectItem>
        ))}
      </SelectContent>
    </Select>
  );
}
```
