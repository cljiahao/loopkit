# superpowers

## Purpose

Archive of loopkit's spec-then-plan development history — a design doc and
an implementation plan per feature/phase, in chronological (dated-filename)
order.

## Contents

- `plans/`
- `specs/`

## Connectivity

Each `specs/` design doc generally has a same-slug counterpart in `plans/`
(design doc `<slug>-design.md` pairs with plan `<slug>.md`) — the design
captures the "what/why", the plan captures the step-by-step "how".
